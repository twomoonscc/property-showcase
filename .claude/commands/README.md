# commands

## code-review
Reviews uncommitted code changes on the current branch by coordinating accessibility and code quality subagents to provide a unified feedback report and action plan.

## commit-message
Analyzes staged git changes to generate a concise, emoji-prefixed commit message that explains the "why" behind the changes.

## component
Generates a new UI component using Test-Driven Development (TDD), creating tests, the component files, and a preview entry.

## spec
Automates the creation of a new feature specification and git branch from a short description, ensuring a clean working directory before starting.

## spec-with-figma-agent
An enhanced version of the spec command that also extracts design specifications from a Figma link to inform the feature documentation.
